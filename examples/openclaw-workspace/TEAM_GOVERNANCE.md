# TEAM_GOVERNANCE.md (Example)

## Agent Orchestration Pipeline

Every task flows through this pipeline:

User (creates) → Leo (routes) → Dana (executes) → Leo (approves) → Done

---

## Agent Roles

### Leo
- **Role**: Governance / Management
- **Responsibility**: Routing tasks and final approval.

### Dana
- **Role**: Pipeline / Execution
- **Responsibility**: Functional implementation.

### Sam
- **Role**: Automation / Checks
- **Responsibility**: Recurring system monitoring.
